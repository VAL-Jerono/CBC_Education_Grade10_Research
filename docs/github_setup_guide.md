# How to Set Up Your GitHub Repository
## Step-by-step for First-Time GitHub Users

---

## Step 1: Create a GitHub Account
1. Go to **https://github.com**
2. Click **Sign Up**
3. Use your academic/professional email
4. Choose a username like: `yourname-research` or `cbc-kenya-research`
5. Verify your email

---

## Step 2: Create a New Repository
1. Once logged in, click the **+** icon (top right) → **New repository**
2. Fill in:
   - **Repository name:** `cbc-teacher-preparedness-kenya`
   - **Description:** `Research repository: CBC teacher preparedness in Kenya Senior School Grade 10-12`
   - **Visibility:** Public (makes it citable and shareable) OR Private (if not ready to share)
   - ✅ Check **Add a README file**
3. Click **Create repository**

---

## Step 3: Upload These Files
1. In your new repo, click **Add file** → **Upload files**
2. Upload all files from this repository folder in their structure:
   ```
   README.md
   data/literature_matrix.csv
   data/themes_frequency.csv
   analysis/thematic_analysis.md
   analysis/gap_analysis.md
   analysis/key_statistics.md
   literature/annotated_bibliography.md
   docs/conceptual_framework.md
   docs/research_questions.md
   docs/github_setup_guide.md
   ```
3. In the **Commit changes** box, write: `Initial commit: literature synthesis and gap analysis`
4. Click **Commit changes**

---

## Step 4: Make It Citable (Optional but Recommended)
1. Go to **https://zenodo.org**
2. Link your GitHub account
3. Create a release in GitHub → Zenodo automatically gives it a **DOI**
4. Now you can cite this repo in your proposal: *"Data and analysis available at doi.org/..."*

---

## Step 5: Keep Updating as You Work
Every time you add new sources, update the CSV and markdown files, then upload again (or use GitHub Desktop for easier syncing).

---

## Pro Tips
- **GitHub Desktop** (desktop.github.com) makes uploading easier than the website
- Add a `LICENSE` file — choose **CC BY 4.0** for academic open access
- Use **Issues** to track your own to-do list (e.g. "Find full text of Mogere 2025")
- Use **Projects** as a Kanban board for your research phases
